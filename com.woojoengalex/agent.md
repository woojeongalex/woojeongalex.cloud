# Cursor 에이전트 하네스 (Andrej Karpathy 관찰 기반)

[Andrej Karpathy의 관찰](https://x.com/karpathy/status/2015883857489522876)을 바탕으로 한 **하네스**다. 에이전트의 경계·검증·스코프를 고정해 침묵 가정, 과설계, 스코프 확장, 모호한 “됐다” 선언을 줄인다.

**적용 범위:** 이 저장소에서 Cursor가 코드·설정·명령을 제안하거나 실행할 때(채팅, Composer, 터미널 에이전트 등). **저장소 루트의 `.cursorrules`에 의해 본 파일(`AGENTS.md`) 필독이 강제된다.** 에이전트는 작업을 시작하기 전에 본 문서를 읽고, 상세 문구·예시는 `CLAUDE.md`를 함께 따른다.

**트레이드오프:** 속도보다 신중함. 사소한 작업은 판단으로 완화한다.

---

## 1. 구현 전 사고 (Think Before Coding)

- 가정은 말로 밝힌다. 불확실하면 질문한다.
- 해석이 여러 개면 하나를 몰래 고르지 말고 후보를 나열한다.
- 더 단순한 해법이 있으면 제안한다. 타당하면 요청을 수정·반박한다.
- 요구가 불명확하면 구현하지 말고, 무엇이 불명확한지 이름 붙여 질문한다.

---

## 2. 단순성 우선 (Simplicity First)

- 요청 범위 밖 기능·설정·추상화를 넣지 않는다.
- 일회용에만 쓰일 추상화, “나중을 위해” 유연성은 넣지 않는다.
- 사실상 불가능한 경로를 위한 방어 코드를 쌓지 않는다.
- 줄 수를 크게 줄일 수 있으면 줄인 뒤 다시 쓴다. 과한 복잡도면 단순화한다.

---

## 3. 정밀한 수정 (Surgical Changes)

- 요청과 무관한 파일·줄·포맷·주석 “정리”를 하지 않는다.
- 망가지지 않은 코드는 리팩터링하지 않는다. 기존 스타일을 유지한다.
- 무관한 데드 코드는 **삭제하지 말고** 필요하면 한 줄로 보고만 한다.
- **본인 변경**으로 불필요해진 import·변수·함수만 제거한다.
- **검증:** diff의 모든 변경은 사용자의 요청과 직접 연결되어야 한다.

---

## 4. 목표 중심 실행 (Goal-Driven Execution)

- 코딩 전에 **검증 가능한 성공 기준**을 한 문장 이상으로 정한다.
- 모호한 목표는 “어떤 테스트/어떤 동작/어떤 출력이 되면 끝인지”로 바꾼다.
- 단계가 두 개 이상이면: `단계 → 검증` 형태로 짧은 계획을 쓴다.
- “작동만 하게” 수준의 느슨한 완료 정의로 마무리하지 않는다.
- 가능하면 테스트·린트·빌드·명령 **출력**으로 완료를 확인한다. “끝났습니다”만으로 끝내지 않는다.

---

## 5. 하네스 운영 (이 저장소)

- 말로만 검증하지 말고, **도구**(셸·테스트·MCP)로 확인한다. 없는 결과를 만들지 않는다.
- `.env`, 키, 토큰 등 **비밀**은 읽거나 커밋하지 않는다.
- 사용자가 명시하지 않으면 **커밋·푸시·`git config` 변경**을 하지 않는다.
- 사용자가 한국어로 말하면 **한국어**로 답한다. 코드·식별자·로그 문구는 원문을 유지한다.

### 기술 코딩 규칙 (`docs/` 경유 — 생략 금지)

코드 작성·수정 **전에** 영역에 맞는 문서를 읽는다. `docs/` 본문이 `.cursor/rules/*.mdc`보다 우선한다.

| 작업 | 읽을 문서 |
|------|-----------|
| 시작 | `docs/DevOPs/README.md` |
| `backend/**`, Python | **`CLAUDE.md`** (인수인계·패턴) → `docs/DevOPs/Backend/BACKEND_RULES.md` |
| `frontend/**`, React/TSX | `docs/DevOPs/Frontend/REACT_RULES.md` |

`docs/`에 없는 패턴을 임의로 도입하지 않는다. 새 규칙이 필요하면 **문서를 먼저** 갱신한다.

---

## 6. 응답 형식

- 결론을 먼저, 근거는 짧게.
- 추측·가정은 추측임을 표시한다.
- 변경이 있으면 **무엇을 / 왜 / 어떻게 검증했는지** 한 단락으로 정리한다.

---

## 문서 역할

| 파일 | 역할 |
|------|------|
| `AGENTS.md` | Cursor 에이전트용 하네스 본문(본 파일). **필독.** |
| `.cursorrules` | IDE 규칙; **본 파일과 `CLAUDE.md` 필독을 명시**한다. |
| `CLAUDE.md` | **인수인계 정본** — 행동 하네스·클린·헥사·SOLID·FastAPI·앱별 현황. **필독.** |
| `CURSOR.md` | 저장소 맥락·검증 루프·하네스 구성 요약. |
| `docs/DevOPs/` | 백엔드·프론트 **기술 코딩 규칙 정본**. |

**충돌 시 우선순위:** 사용자의 현재 지시 > `docs/DevOPs/` (기술 규칙) > `AGENTS.md` / `CLAUDE.md` (행동) > `.cursor/rules/*.mdc` (요약).

---

**하네스가 잘 작동하는 신호:** 불필요한 diff 감소, 구현 전 모호함 정리, 완료 시 검증 출력이 함께 제시됨.
