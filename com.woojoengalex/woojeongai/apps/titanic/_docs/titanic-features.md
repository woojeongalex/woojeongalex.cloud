# 타이타닉 생존자 예측 데이터셋 설명서 (Dataset Description)

## 📌 1. 데이터 개요 (Overview)
본 데이터셋은 타이타닉호 침몰 사건의 탑승객 데이터를 바탕으로 알고리즘을 학습시켜 각 탑승객의 생존 여부를 예측하기 위해 두 개의 그룹으로 분할되어 있습니다.

* **학습 데이터셋 (`train.csv`)**: 머신러닝 모델을 빌드하고 학습시키는 데 사용됩니다. 각 탑승객의 생존 여부 정답 값(Ground Truth)이 제공됩니다. 탑승객의 성별, 티켓 등급 등의 '피처(Features)'를 기반으로 학습을 진행하며, 예측 성능을 높이기 위한 **피처 엔지니어링(Feature Engineering)**의 대상이 됩니다.
* **테스트 데이터셋 (`test.csv`)**: 학습된 모델이 보지 못한 데이터(Unseen Data)에 대해 얼마나 잘 작동하는지 검증하는 데 사용됩니다. 생존 여부 정답 값이 제공되지 않으며, 학습된 모델을 통해 각 탑승객의 생존 여부를 예측해야 합니다.
* **제출 파일 예시 (`gender_submission.csv`)**: 오직 여성 탑승객만 생존했다고 가정한 단순 예측 파일로, 최종 제출해야 하는 결과 파일의 올바른 정답 포맷 예시를 제공합니다.

## 📊 2. 데이터 사전 (Data Dictionary)

| 변수명 (Variable) | 정의 (Definition) | 키 값 설명 (Key / Values) |
| :--- | :--- | :--- |
| **`survived`** | 생존 여부 (타깃 변수) | `0` = 사망 (No), `1` = 생존 (Yes) |
| **`pclass`** | 티켓 등급 (사회·경제적 지위 수준) | `1` = 1등석 (Upper), `2` = 2등석 (Middle), `3` = 3등석 (Lower) |
| **`gender`** | 성별 | `male` = 남성, `female` = 여성 |
| **`age`** | 탑승객의 나이 (연령) | fractional (1세 미만은 소수점), estimated (추정 나이는 `xx.5` 형태) |
| **`sib_sp`** | 함께 탑승한 형제자매 및 배우자의 수 | 동반한 형제, 자매, 의붓형제, 의붓자매, 남편, 아내의 총합 |
| **`parch`** | 함께 탑승한 부모 및 자녀의 수 | 동반한 어머니, 아버지, 딸, 아들, 의붓딸, 의붓아들의 총합 |
| **`ticket`** | 티켓 번호 | 탑승객 고유의 티켓 문자열/숫자 |
| **`fare`** | 여객 운임 (티켓 요금) | 탑승객이 지불한 요금 수치 |
| **`cabin`** | 선실 번호 | 탑승객이 머물렀던 구역 및 객실 번호 |
| **`embarked`** | 탑승 항구 (탑승지) | `C` = 셰르부르(Cherbourg), `Q` = 퀸즈타운(Queenstown), `S` = 사우샘프턴(Southampton) |

## 🔍 3. 변수별 세부 분석 노트 (Variable Notes)

### 🏢 티켓 등급 (`pclass`)
* 탑승객의 사회적, 경제적 지위(SES)를 나타내는 대리 변수(Proxy) 역할을 합니다.
* `1st (1등석)`은 최상류층, `2nd (2등석)`은 중산층, `3rd (3등석)`은 하위 노동자층을 의미하며 생존율과 매우 밀접한 연관성을 가질 가능성이 높습니다.

### 👶 나이 (`age`)
* 나이가 1세 미만인 유아의 경우 소수점 형태로 표현됩니다. (예: `0.42`)
* 정확하지 않아 추정된 탑승객의 나이는 값의 끝자리가 항상 `.5` 형태로 저장되어 있습니다. (예: `28.5`)

### 👫 형제자매 / 배우자 수 (`sibsp`)
* 데이터셋에서 정의하는 가족 관계 규격은 다음과 같습니다.
    * **형제자매(Sibling)**: 남동생, 여동생, 형, 누나, 오빠, 의붓형제, 의붓자매를 포함합니다.
    * **배우자(Spouse)**: 남편과 아내만 인정합니다. (단, 내연녀나 약혼자는 가족 집계에서 제외되었습니다.)

### 👨‍👩‍👧‍👦 부모 / 자녀 수 (`parch`)
* 데이터셋에서 정의하는 직계가족 관계 규격은 다음과 같습니다.
    * **부모(Parent)**: 어머니, 아버지를 포함합니다.
    * **자녀(Child)**: 딸, 아들, 의붓딸, 의붓아들을 포함합니다.
    * *주의사항*: 부모 없이 오직 보모(Nanny)하고만 함께 여행한 아이들의 경우, 부모가 동승하지 않았으므로 `parch=0`으로 기록되어 있습니다.

## 💡 4. 클로드를 위한 피처 엔지니어링 추천 유도 가이드 (Feature Engineering Brainstorming)

클로드가 이 문서를 읽고 새로운 파생 변수를 발굴할 수 있도록 도메인 지식 기반의 힌트 구조를 남겨둡니다.

1.  **가족 규모 통합 변수 (`FamilySize`)**: `sibsp`와 `parch`를 더하고 자기 자신(+1)을 더해 탑승객의 총 가족 규모 파생 변수 생성 가능.
2.  **호칭 추출 변수 (`Title`)**: `ticket`이나 탑승객 이름 문자열에서 'Mr.', 'Mrs.', 'Master', 'Miss' 등의 호칭을 추출하여 사회적 지위나 결혼 여부, 미성년 여부를 세분화 정제 가능.
3.  **선실 구역 분리 (`Cabin_Init`)**: `cabin` 번호의 첫 글자(알파벳)만 추출하여 배의 어느 구역(Deck)에 위치했는지 범주화 가능.
4.  **1인 탑승 여부 (`IsAlone`)**: `FamilySize`가 1명인 탑승객을 바이너리(0 또는 1) 변수로 분리하여 동반 가족 유무에 따른 생존율 유도 가능.
