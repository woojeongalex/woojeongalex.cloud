# 🚢 타이타닉 생존자 예측 AI 프로젝트 가이드

비전문가의 눈높이에 맞춰 전체 흐름을 **'데이터 준비 ➡️ 정제(전처리) ➡️ AI 학습 ➡️ 결과 제출'**의 4단계로 구조화했습니다.

이 가이드는 타이타닉호 침몰 사고의 탑승객 데이터를 활용하여, 특정 승객이 "생존했을지(Survived)" 아니면 "사망했을지"를 AI 모델에게 학습시키고 예측하는 전체 과정을 담고 있습니다.

---

## 📅 1. 데이터 개요 및 준비 (Overview)

AI를 학습시키기 위해 데이터는 크게 두 가지 그룹으로 나뉩니다.

* **학습용 데이터 (`train.csv`)**
  * AI를 훈련시키는 교과서입니다.
  * 승객들의 개인 정보(성별, 나이, 등급 등)와 함께 실제 생존 여부(정답, Ground Truth)가 포함되어 있습니다.
* **시험용 데이터 (`test.csv`)**
  * AI가 얼마나 잘 맞추는지 확인하는 시험지입니다.
  * 승객들의 개인 정보는 들어있지만, 생존 여부(정답)는 숨겨져 있습니다. AI가 이 데이터의 생존 여부를 예측해야 합니다.

### 📊 데이터의 종류 (변수/피처/컬럼) 알아보기
승객의 정보는 성격에 따라 4가지 데이터 측정 척도로 분류되어 처리됩니다.

| 데이터 대분류 | 세부 척도 (Type) | 타이타닉 데이터 예시 | 설명 |
| :--- | :--- | :--- | :--- |
| **Categorical**<br>(범주형: 그룹으로 묶임) | **Nominal (명목형)** | 성별(Sex), 승선항(Embarked), 이름(Name) | 순서가 없고 단순히 이름이나 종류를 나타내는 데이터 |
| | **Ordinal (순서형)** | 객실 등급(Pclass: 1등석, 2등석, 3등석) | 데이터 사이에 명확한 순위나 서열이 있는 데이터 |
| **Quantitative**<br>(수치형: 숫자로 계산 가능) | **Interval (간격형)** | 본 데이터셋에는 해당 없음 | 절대적 기준점(0)은 없으나 일정한 측정 구간을 가진 데이터 |
| | **Ratio (비율형)** | 나이(Age), 요금(Fare), 동반자 수(SibSp, Parch) | 기준점 0이 존재하며, '2배 많다' 등의 비율 계산이 가능한 데이터 |

## 🛠️ 2. 데이터 전처리 단계 (Data Preprocessing)

AI는 텍스트(글자)나 빈 데이터(NaN)를 그대로 이해하지 못합니다. 따라서 컴퓨터가 이해할 수 있는 정제된 숫자 형태로 변환하는 과정(전처리)이 필요합니다.

### 🔄 단계별 전처리 과정

1. **불필요한 정보 버리기 (`drop_feature`)**
   * 생존 분석에 큰 영향이 없거나 방해가 되는 티켓 번호(Ticket), 객실 번호(Cabin), 동반자 수(SibSp, Parch) 등의 정보를 데이터셋에서 삭제합니다.
2. **이름에서 호칭 추출 및 수치화 (`Title`)**
   * 승객 이름(예: Braund, Mr. Owen Harris)에서 Mr, Mrs, Miss 같은 호칭만 쏙 빼냅니다.
   * 드문 호칭이나 귀족 호칭은 Rare, Royal로 묶어준 뒤, 컴퓨터가 이해하도록 숫자 매핑을 진행합니다.
   * `Mr ➡️ 1`, `Ms ➡️ 2`, `Mrs ➡️ 3`, `Master ➡️ 4`, `Royal ➡️ 5`, `Rare ➡️ 6`
3. **성별 및 승선항 숫자 변환 (`Nominal`)**
   * 글자로 된 성별과 승선항을 기계가 인지할 수 있는 숫자로 바꿉니다.
   * 성별: `남성 ➡️ 0`, `여성 ➡️ 1`
   * 승선항: `S ➡️ 1`, `C ➡️ 2`, `Q ➡️ 3`
4. **비어있는 나이 채우고 구역 나누기 (`Age Ratio`)**
   * 나이 데이터가 비어있는 승객은 임시로 -0.5 처리를 해둔 뒤, 연령대별 구간(bins)을 나누어 그룹화합니다.
   * 아기(Baby), 어린이(Child), 청소년(Teenager), 학생(Student), 청년(Young Adult), 성인(Adult), 노인(Senior)으로 분류한 후, 이 역시 0~7까지의 숫자로 변환합니다.
5. **객실 등급 확인 및 요금 그룹화 (`Pclass, Fare Ratio`)**
   * 객실 등급(1, 2, 3등석)의 순서형 데이터를 정돈합니다.
   * 승객이 낸 요금(Fare)도 금액에 따라 Low(낮음), Mid(보통), High(높음) 등의 구간을 나누어 숫자로 바꿉니다.

## 🤖 3. 모델링 및 AI 학습 단계 (Modeling & Learning)

데이터 정제가 끝났다면 이제 본격적으로 다양한 AI 알고리즘들에게 기출문제(`train.csv`)를 풀게 하여 공부를 시킵니다.

### 👨‍🏫 다양한 AI 알고리즘 교사들
하나의 알고리즘만 쓰지 않고, 여러 명의 AI 알고리즘에게 학습을 시켜본 후 가장 똑똑한 결과를 내는 녀석을 선택합니다.

* **결정 트리 (Decision Tree):** 스무고개 방식으로 데이터를 분류하는 알고리즘
* **랜덤 포레스트 (Random Forest):** 여러 개의 결정 트리를 만들어 다수결로 결과를 정하는 알고리즘
* **나이브 베이즈 (Naive Bayes):** 확률을 기반으로 생존 여부를 예측하는 알고리즘
* **KNN (최근접 이웃):** 나와 비슷한 조건을 가진 주변 이웃 승객들의 결과를 보고 판단하는 알고리즘
* **SVM (서포트 벡터 머신):** 데이터들 사이에 최적의 경계선을 그어 생존과 사망을 나누는 알고리즘

### 📈 검증 및 정확도 확인
AI가 학습을 마치면 모의고사를 치르게 하여 각 알고리즘별 검증 정확도(Accuracy)를 출력합니다.
* *출력 예시:* `"사이킷런 알고리즘 정확도: 70%"`

---

## 🏁 4. 후처리 및 최종 제출 (Postprocessing & Submit)

모든 학습과 검증이 끝나면 가장 우수한 성적을 거둔 AI 모델을 채택합니다.

1. **후처리 (Postprocessing):** 예측된 결과를 제출 양식에 맞게 최종 정돈합니다.
2. **최종 예측 및 제출 (Submit):** 정답이 없었던 시험지 데이터(`test.csv`)를 AI에게 건네주어 생존 여부를 예측하게 한 뒤, 그 결과를 파일로 저장하여 제출(Kaggle 등)하며 프로젝트가 마무리됩니다.
