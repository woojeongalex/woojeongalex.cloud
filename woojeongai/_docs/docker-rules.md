# docker-rules.md — DB·백엔드 중복 생성 방지 하네스

> 적용 범위: `docker-compose.yaml` 서비스, `woojeongai/apps/<앱>` 백엔드, DB(테이블·스키마) 관련 신규 생성 요청 전체

---

## 0. 핵심 규칙

**사용자가 "DB 만들어줘" / "백엔드 새로 만들어줘" 라고 지시해도, 동일하거나 유사한 것이 이미 존재하면 곧바로 생성하지 않는다.**

1. 먼저 기존 리소스를 **체크**한다 (§2 절차).
2. 기존 것이 있으면 생성을 멈추고, 발견한 내용을 사용자에게 **보고**한다.
3. 사용자의 **명시적 승인**(재생성 / 덮어쓰기 / 별도 이름으로 생성 등)을 받은 뒤에만 진행한다.
4. 승인 없이 임의로 재생성·덮어쓰기·중복 생성하지 않는다.

> "일단 만들고 나중에 물어본다"는 이 하네스에서 금지된다. 체크 → 보고 → 승인 → 생성 순서를 반드시 지킨다.

---

## 1. 왜 필요한가

- `docker-compose.yaml`에는 `pgvector`(Postgres), `redis`, `neo4j`, `n8n`, `backend`(FastAPI) 등 이름이 고정된 컨테이너·볼륨이 이미 정의되어 있다.
- 같은 이름으로 서비스/컨테이너/볼륨을 다시 만들면 기존 데이터 볼륨(`pgvector_data`, `redis_data`, `neo4j_data` 등)을 덮어쓰거나 포트 충돌이 발생할 수 있다.
- 백엔드도 `woojeongai/apps/<앱명>/` 구조가 앱별로 이미 존재할 수 있어, 같은 도메인을 중복 스캐폴딩하면 §3(클린 아키텍처) 구조가 깨지거나 라우터·테이블이 충돌한다.

---

## 2. 생성 전 체크 절차

### 2.1 DB(컨테이너·스키마·테이블) 요청 시

```bash
# 1) 이미 떠 있는 컨테이너/서비스 확인
docker compose ps
docker ps -a --format "table {{.Names}}\t{{.Image}}\t{{.Status}}"

# 2) docker-compose.yaml에 동일 서비스명/컨테이너명/포트가 있는지 확인
grep -nE "container_name|services:|ports:" docker-compose.yaml

# 3) 요청받은 테이블/스키마가 이미 있는지 ORM·마이그레이션 이력 확인
grep -rn "<테이블명>" woojeongai/apps/*/adapter/outbound/orm/
ls woojeongai/alembic/versions/ | grep -i "<테이블명>"
```

### 2.2 백엔드(앱·서비스) 요청 시

```bash
# 1) 동일/유사 이름의 앱이 이미 있는지 확인
ls woojeongai/apps/

# 2) main.py에 이미 등록된 router인지 확인
grep -n "include_router" woojeongai/main.py

# 3) 동일 역할의 Director/UseCase가 이미 있는지 확인
find woojeongai/apps -iname "*<도메인명>*"
```

---

## 3. 판단 기준

| 상황 | 행동 |
|------|------|
| 동일 이름의 컨테이너/서비스/앱이 이미 존재 | **생성 중단** → 발견 내용 보고 → 승인 대기 |
| 이름은 다르지만 목적·역할이 겹침 (예: 같은 DB에 대한 두 번째 Postgres 컨테이너) | **생성 중단** → 중복 가능성 보고 → 승인 대기 |
| 기존 것이 없고, grep/ls 결과가 명확히 비어 있음 | 정상 생성 진행 (승인 불필요) |

승인 없이 진행해도 되는 경우는 **기존 리소스가 전혀 없다고 확인된 경우뿐**이다. 애매하면 승인을 받는다.

---

## 4. 보고 형식 (승인 요청 시)

사용자에게 승인을 구할 때는 아래 정보를 포함한다:

- 발견한 기존 리소스 (컨테이너명 / 서비스명 / 앱 경로 / 테이블명)
- 위치 (파일 경로 또는 `docker compose ps` 출력)
- 선택지: (a) 기존 것 재사용, (b) 기존 것 덮어쓰기/재생성, (c) 별도 이름으로 신규 생성

---

## 5. 안티패턴 (하지 말 것)

```bash
# ❌ 기존 컨테이너/볼륨 확인 없이 바로 docker-compose.yaml에 동일 이름 서비스 추가
# ❌ apps/ 디렉터리 확인 없이 같은 도메인 앱을 새로 스캐폴딩
# ❌ "사용자가 만들어달라고 했으니까" 를 이유로 체크 절차를 생략
# ❌ 승인 요청 없이 기존 볼륨(pgvector_data 등)을 덮어쓰는 재생성
```
