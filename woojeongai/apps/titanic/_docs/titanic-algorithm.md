주신 내용과 추가 요청 사항(머신러닝 10대 알고리즘, 변수의 개념 정의, 표준화/정규화 전처리 팁)을 결합하여, 타이타닉 경진대회 분석 및 일반적인 머신러닝 기초 학습에 모두 활용할 수 있도록 **하나의 완성된 구조화된 마크다운(MD) 문서**로 정리했습니다.

# 🚢 캐글 타이타닉으로 배우는 머신러닝 알고리즘 및 전처리 가이드

캐글(Kaggle)의 타이타닉 경진대회는 정형 데이터 분류 문제의 고전입니다. 본 문서에서는 타이타닉 생존자 예측에 효과적인 핵심 알고리즘 10가지와 필수 데이터 전처리 개념을 정리합니다.

---

## 1. 머신러닝 핵심 변수 개념 이해

모델을 학습시키기 전, 데이터셋을 구성하는 변수의 개념을 명확히 정의해야 합니다.

* **특성 변수 (Feature)**
* **동의어:** 독립변수, 외생변수, $X$ 변수
* **설명:** 모델이 학습할 입력값입니다. 타이타닉 예제에서는 `Pclass(객실 등급)`, `Sex(성별)`, `Age(나이)`, `Fare(요금)` 등이 이에 해당합니다.


* **목적 변수 (Target)**
* **동의어:** 종속변수, 내생변수, $Y$ 변수
* **설명:** 모델이 예측해야 하는 결과값입니다. 타이타닉 예제에서는 `Survived(생존 여부: 0 또는 1)`가 목적 변수입니다.



---

## 2. 피처 변환 (Feature Transformation): 표준화 vs 정규화

수치형 변수의 단위를 맞추기 위한 스케일링(Scaling) 기법으로, 데이터의 분포와 이상치(Outlier) 유무에 따라 선택합니다.

* **표준화 (Standardization)**
* **개념:** 데이터를 평균이 0, 표준편차가 1인 표준정규분포 형태로 변환합니다.
* **선택 기준:** 데이터에 **아웃라이어(이상치)가 존재할 때** 유리합니다. 이상치의 영향을 덜 받기 때문입니다.


* **정규화 (Normalization / Min-Max Scaling)**
* **개념:** 데이터의 최솟값을 0, 최댓값을 1로 하여 모든 값을 0과 1 사이로 변환합니다.
* **선택 기준:** 아웃라이어가 없고 데이터의 분포가 비교적 균일할 때 효과적입니다.



---

## 3. 타이타닉 필수 반영 머신러닝 알고리즘 TOP 10

타이타닉 생존자 예측(분류) 및 머신러닝 기초 학습을 위해 마스터해야 할 10대 알고리즘입니다.

| 순위 | 알고리즘 명칭 | 주요 특징 | 타이타닉 적용 및 추천 이유 |
| --- | --- | --- | --- |
| **1** | **XGBoost** | 그래디언트 부스팅 기반 고성능 모델 | 강력한 규제(Regularization) 기능으로 과적합을 방지하며 리더보드 최상위권 점수를 낼 때 필수적입니다. |
| **2** | **Random Forest** | 다수의 결정 트리를 결합하는 배깅 방식 | 데이터 노이즈에 강하며, 하이퍼파라미터 튜닝 없이도 매우 안정적인 Baseline 성능을 보장합니다. |
| **3** | **LightGBM** | 리프 중심(Leaf-wise) 트리 분할 방식 | 대용량 처리에 특화되어 성능이 우수하고 속도가 빨라 복잡한 피처 조합을 실험할 때 유용합니다. |
| **4** | **CatBoost** | 범주형 데이터 처리에 최적화된 부스팅 | 성별(`Sex`), 승선지(`Embarked`) 등 범주형 피처가 많은 타이타닉 데이터셋을 별도 인코딩 없이 최적화합니다. |
| **5** | **Logistic Regression** | 선형 관계를 기반으로 한 이진 분류 모델 | 타이타닉 생존 여부(0/1)를 예측하는 기본 모델로, 각 피처가 생존에 미치는 영향력을 직관적으로 해석하기 좋습니다. |
| **6** | **Decision Tree** | 나무 가지치기 형태의 직관적인 규칙 기반 모델 | 데이터의 흐름과 분류 기준을 시각화하여 확인하기 좋으나, 과적합 위험이 있어 튜닝이 필요합니다. |
| **7** | **SVM (Support Vector Machine)** | 마진을 최대화하는 최적 결정 경계 탐색 | 데이터 전처리(표준화 등)가 정교하게 이루어졌을 때, 변수 간의 복잡한 비선형 관계를 파악하는 데 유수합니다. |
| **8** | **KNN (K-Nearest Neighbors)** | 주변의 가장 가까운 $K$개 이웃 기준 분류 | 승객 간의 유사도(나이, 요금, 객실 등)를 기반으로 작동하며, 단순하지만 직관적인 결과를 줍니다. |
| **9** | **Naive Bayes** | 베이즈 정리를 이용한 조건부 확률 기반 분류 | 모든 특성이 독립적이라고 가정하며, 계산이 매우 빠르고 희소한 데이터셋에서도 준수한 성능을 냅니다. |
| **10** | **K-Means & PCA** | 비지도 학습 기반 군집화 및 차원 축소 | 분류 모델은 아니지만, 승객 그룹을 클러스터링(`K-Means`)하거나 수많은 피처를 압축(`PCA`)하여 새로운 파생 변수를 만들 때 보조적으로 활용됩니다. |

## 4. 정확도 향상을 위한 핵심 실전 전략

알고리즘 선정만큼 중요한 세 가지 실전 테크닉입니다.

### ✅ 피처 엔지니어링 (Feature Engineering)

알고리즘 성능보다 더 중요한 것은 데이터 가공입니다.

* **Title 추출:** 이름(`Name`) 문자열에서 'Mr', 'Miss', 'Master' 등을 추출하여 사회적 지위 및 성별·연령대를 보완합니다.
* **Family Size:** 형제자매/배우자 수(`SibSp`)와 부모/자녀 수(`Parch`)를 합쳐 가족 규모 변수를 생성합니다. 단독 탑승 여부(`IsAlone`)도 유의미한 피처가 됩니다.
* **Age Binning:** 나이를 단순 숫자가 아닌 유아, 청소년, 청년, 장년, 노년 등 연령대 그룹으로 묶어 패턴을 단순화합니다.

### ✅ 앙상블(Ensemble) 기법 활용

* **Soft Voting:** 개별 모델(XGBoost, Random Forest 등)의 예측 확률을 평균 내어 최종 결과를 도출합니다.
* **Stacking:** 개별 모델들의 예측값을 다시 입력 데이터($X$)로 삼아 최종 메타 모델이 다시 학습하는 방식입니다.

### ✅ 하이퍼파라미터 튜닝

`GridSearchCV`나 `Optuna` 같은 라이브러리를 사용하여 모델별 최적의 파라미터(트리 깊이, 학습률, 추정기 개수 등)를 찾는 과정이 필수적입니다.
