## 1. 모델 선택 — OpenPose 대신 ViTPose (RTX 3050 최적)

원서의 OpenPose는 C++ 기반으로 Python 통합이 복잡하고, RTX 3050 8GB에서 다중 인물 추론 시 VRAM 한계.
아래 모델 중 하나를 선택한다 (기본값: ViTPose-Small).

| 모델 | VRAM 추론 | VRAM 파인튜닝(QLoRA) | COCO mAP | 지원 |
|------|-----------|---------------------|----------|------|
| **ViTPose-Small** | ~1.5GB | ~3GB (배치 8) | 73.8 | HuggingFace ✅ 기본값 |
| ViTPose-Base | ~3GB | ~5GB (배치 4) | 75.8 | HuggingFace ✅ |
| MediaPipe Pose | ~0.5GB | ❌ 파인튜닝 불가 | — | Google ✅ |
| RTMPose-m | ~2GB | ~4GB (배치 4) | 75.3 | mmpose |
| OpenPose (원서) | ~3GB | ❌ C++ 기반 | 61.8 | caffe |

> **QLoRA 파인튜닝 목적이면 ViTPose-Base 추천** — Small 대비 mAP +2, 8GB에서 배치 4 처리 가능.
> **빠른 추론만 필요하면 MediaPipe Pose** — CPU에서도 실시간 동작, 파인튜닝 불필요.

## 2. 구현할 기능

ViTPose-Small로 이미지에서 인물의 17개 관절(keypoint)을 추정하고,

- 관절 좌표와 신뢰도를 **Neo4j Graph DB** 에 노드로 저장한다.
- 관절 연결선(skeleton)이 그려진 시각화 이미지를 **S3** 에 업로드한다.
- FastAPI `POST /api/star-craft/pose/estimate` 로 이미지를 수신한다.
- (선택) `POST /api/star-craft/pose/train` 으로 커스텀 데이터셋 파인튜닝.

### Step 5 — Tool 함수 (CPU/GPU-bound)

**`app/tools/vitpose_estimate_tool.py`**
```
- 싱글턴 패턴:
    _extractor = None
    _model = None
    _model_lock = threading.Lock()

- def _get_model_and_extractor() -> tuple:
    global _extractor, _model
    with _model_lock:
        if _model is None:
            from transformers import ViTPoseForPoseEstimation, AutoImageProcessor
            _extractor = AutoImageProcessor.from_pretrained(
                nielsr/vitpose-small-simple
            )
            _model = ViTPoseForPoseEstimation.from_pretrained(
                nielsr/vitpose-small-simple
            )
            device = cuda if torch.cuda.is_available() else cpu
            _model.to(device)
            _model.eval()
    return _extractor, _model

- def estimate_pose(image_bytes: bytes, threshold: float) -> PoseVO:
    extractor, model = _get_model_and_extractor()
    image = Image.open(io.BytesIO(image_bytes)).convert(RGB)
    W, H = image.size

    # ViTPose는 바운딩박스를 입력으로 받음
    # 전처리: 이미지 전체를 단일 박스로 처리 (단순화)
    boxes = [[0, 0, W, H]]
    inputs = extractor(image, boxes=[boxes], return_tensors=pt).to(model.device)

    with torch.no_grad():
        outputs = model(**inputs)

    # 후처리: keypoint 좌표와 신뢰도 추출
    pose_results = extractor.post_process_pose_estimation(
        outputs, boxes=[boxes]
    )[0]   # 첫 번째 이미지 결과

    persons = []
    for person_id, person in enumerate(pose_results):
        keypoints_raw = person[keypoints]   # (17, 2) 절대 좌표
        scores_raw    = person[scores]       # (17,) 신뢰도

        kps = []
        for i, (coord, score) in enumerate(zip(keypoints_raw, scores_raw)):
            kps.append(Keypoint(
                name=COCO_KEYPOINTS[i],
                x=float(coord[0]) / W,
                y=float(coord[1]) / H,
                confidence=float(score),
            ))

        visible = sum(1 for k in kps if k.confidence >= threshold)
        persons.append(PersonPose(person_id=person_id, keypoints=kps, visible_count=visible))

    return PoseVO(
        persons=persons,
        person_count=len(persons),
        image_width=W,
        image_height=H,
    )

- CPU/GPU-bound → def estimate_pose(...) — async 절대 금지
```

## 6. 안티패턴 — 하지 말 것

```python
# ❌ 추론 함수에 async 붙이기 (GPU-bound)
async def estimate_pose(image_bytes: bytes): ...

# ❌ Keypoint 절대 좌표를 VO에 그대로 담기 — 이미지 크기 의존성
@dataclass(frozen=True)
class Keypoint:
    x: int   # 절대 픽셀 금지 — 반드시 0.0~1.0 정규화

# ❌ PoseVO를 DTO에 직접 담기 — 직렬화 불가
@dataclass(frozen=True)
class PoseEstimateResponse:
    vo: PoseVO   # 절대 금지 — dict로 변환 후 담을 것

# ❌ 스켈레톤 시각화 툴이 DTO를 받는 경우
def draw_skeleton(response: PoseEstimateResponse): ...   # PoseVO만 받아야 함

# ❌ 3개 포트를 하나로 합치기 (ISP 위반)
class PosePort(ABC):
    def save(...): ...
    async def save_pose(...): ...
```
