## 0. 구현 전 필독

1. `woojeongai/CLAUDE.md` 전문을 읽고 4계층·ISP·DIP·Director 패턴을 숙지한다.
2. **패턴 정본** — 아래 파일을 반드시 먼저 읽어 기존 스타일을 맞춘다.
   - `star_craft/dependencies/yolo_provider.py`
   - `star_craft/adapter/outbound/resource_adapters/yolo/yolo_model_adapter.py`
3. 기존 파일을 **절대 수정하지 않는다**. 신규 파일만 추가한다.
4. 모든 파일 상단에 `from __future__ import annotations` 를 붙인다.
5. 감정 분석 추론은 GPU-bound → `def` (동기), 호출 측에서 `asyncio.to_thread` 위임.
6. 이 에이전트는 **다른 에이전트(이미지·동영상)의 캡션/설명 텍스트**를 입력으로 받아
   감정을 분석하는 멀티에이전트 파이프라인의 NLP 컴포넌트로 설계한다.

## 1. 모델 선택 — klue/roberta-base (한국어 우선)

원서는 영어 Transformer 기반. 이 프로젝트는 한국어·영어 혼용 환경이므로
다국어 또는 한국어 특화 모델을 기본값으로 선택한다.

| 모델 | VRAM 추론 | VRAM 파인튜닝(QLoRA) | SST-2 | 한국어 | 기본값 |
|------|-----------|---------------------|-------|--------|--------|
| **klue/roberta-base** | ~0.8GB | ~2GB (배치 32) | — | ✅ 96.1% | ✅ |
| snunlp/KR-ELECTRA-discriminator | ~0.8GB | ~2GB (배치 32) | — | ✅ 96.0% | — |
| monologg/koelectra-base-v3 | ~0.8GB | ~2GB (배치 32) | — | ✅ 95.5% | — |
| xlm-roberta-base (다국어) | ~1.1GB | ~3GB (배치 16) | 94.8% | ✅ | — |
| distilbert-base-uncased (영어) | ~0.5GB | ~1GB (배치 64) | 91.3% | ❌ | — |

> **한국어만 사용하면 klue/roberta-base**, 한·영 혼용이면 xlm-roberta-base 선택.

## 2. 구현할 기능

Transformer 기반 감정 분류기로 텍스트의 감정을 분석하고,

- 분석 결과(감정 레이블 + 신뢰도)를 **Neo4j Graph DB** 에 노드로 저장한다.
- `POST /api/star-craft/sentiment/analyze` — 단일 또는 배치 텍스트 분석
- `POST /api/star-craft/sentiment/train` — 커스텀 데이터셋으로 파인튜닝
- 멀티에이전트 연동: 이미지 분류 에이전트의 레이블을 텍스트로 변환 후 감정 분석 가능

### Step 1 — Domain Value Object

**`domain/value_objects/sentiment_vo.py`**
```
- frozen dataclass

- SentimentLabel: 열거형(str Enum)
    POSITIVE = positive
    NEGATIVE = negative
    NEUTRAL  = neutral   # 3-class 파인튜닝 시 사용

- SentimentResult:
    label: SentimentLabel
    confidence: float       # 0.0~1.0
    scores: dict[str, float]   # {positive: 0.9, negative: 0.1} 전체 확률

- SentimentVO:
    text: str
    result: SentimentResult
    model_name: str

- __post_init__ 검증:
    confidence 0.0~1.0
    sum(scores.values()) ≈ 1.0 (오차 0.01)
    len(text) > 0
```

### Step 5 — Tool 함수

**`app/tools/sentiment_analyze_tool.py`**
```
- 싱글턴 패턴 (model_name별 캐시):
    _pipelines: dict[str, pipeline] = {}
    _pipeline_lock = threading.Lock()

- def _get_pipeline(model_name: str) -> pipeline:
    with _pipeline_lock:
        if model_name not in _pipelines:
            from transformers import pipeline as hf_pipeline
            _pipelines[model_name] = hf_pipeline(
                text-classification,
                model=model_name,
                tokenizer=model_name,
                device=0 if torch.cuda.is_available() else -1,
                top_k=None,   # 전체 레이블 확률 반환
            )
    return _pipelines[model_name]

- def analyze_sentiment(
      texts: list[str],
      model_name: str,
      max_length: int,
  ) -> list[SentimentVO]:

    pipe = _get_pipeline(model_name)
    raw_results = pipe(texts, truncation=True, max_length=max_length, padding=True)
    # raw_results: [[{label: LABEL_1, score: 0.9}, ...], ...]

    vos = []
    for text, result_list in zip(texts, raw_results):
        scores = {_normalize_label(r[label]): r[score] for r in result_list}
        best = max(scores, key=lambda k: scores[k])
        vos.append(SentimentVO(
            text=text,
            result=SentimentResult(
                label=SentimentLabel(best),
                confidence=scores[best],
                scores=scores,
            ),
            model_name=model_name,
        ))
    return vos

- def _normalize_label(raw: str) -> str:
    # LABEL_1 → positive, LABEL_0 → negative
    # POSITIVE → positive 등 모델별 레이블 정규화
    mapping = {
        LABEL_1: positive, LABEL_0: negative,
        POSITIVE: positive, NEGATIVE: negative,
        긍정: positive, 부정: negative, 중립: neutral,
    }
    return mapping.get(raw.upper(), raw.lower())

- GPU-bound → def analyze_sentiment(...) — async 절대 금지
```

```
- SentimentInteractor(SentimentUseCase) 구현
- __init__: model: SentimentModelPort, graph: SentimentGraphPort

- def analyze(self, cmd: AnalyzeCommand) -> AnalyzeResponse:
    vos = analyze_sentiment(cmd.texts, cmd.model_name, cmd.max_length)
    results = [
        {
            text: vo.text,
            label: vo.result.label.value,
            confidence: vo.result.confidence,
            scores: vo.result.scores,
        }
        for vo in vos
    ]
    return AnalyzeResponse(
        results=results, model_name=cmd.model_name, analyzed_count=len(results)
    )

- def train(self, cmd: TrainCommand) -> TrainResponse:
    tmp_dir, accuracy, f1 = train_sentiment(cmd)
    weights_path = self.model.save(tmp_dir)
    return TrainResponse(
        weights_path=weights_path, epochs=cmd.epochs,
        eval_accuracy=accuracy, eval_f1=f1,
    )
```

## 5. 안티패턴 — 하지 말 것

```python
# ❌ analyze / train에 async 붙이기 (GPU-bound)
async def analyze(self, cmd): ...
async def train(self, cmd): ...

# ❌ pipeline을 요청마다 새로 로드
def analyze(self, cmd):
    pipe = pipeline(text-classification, model=cmd.model_name)  # 매 요청마다 로드 금지

# ❌ SentimentVO를 DTO에 직접 담기
@dataclass(frozen=True)

    vos: list[SentimentVO]   # 절대 금지 — dict 변환 후 담을 것

# ❌ label_map을 코드에 하드코딩

# ❌ 라우터에서 asyncio.to_thread 없이 동기 메서드 직접 호출
```
